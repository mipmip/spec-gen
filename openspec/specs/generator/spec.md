## Requirements

### Requirement: OpenSpecValidatorValidation

The system SHALL validate OpenSpecValidator according to these rules:
- validateSpecStructure() checks for title, purpose, requirements sections
- validateRequirementFormat() enforces RFC 2119 keywords and imperative form
- validateScenarioFormat() ensures proper Given/When/Then structure
- validateConfigYaml() validates config.yaml structure and fields

#### Scenario: ValidateSpecStructure
- **GIVEN** A spec document
- **WHEN** validateSpecStructure() is called
- **THEN** Returns validation result with errors for missing required sections

#### Scenario: ValidateRequirementFormat
- **GIVEN** A requirement string
- **WHEN** validateRequirementFormat() is called
- **THEN** Returns validation result with warnings for non-RFC 2119 keywords

#### Scenario: ValidateScenarioFormat
- **GIVEN** A scenario string
- **WHEN** validateScenarioFormat() is called
- **THEN** Returns validation result with errors for missing GIVEN/WHEN/THEN clauses

#### Scenario: ValidateConfig.yaml
- **GIVEN** A config object
- **WHEN** validateConfigYaml() is called
- **THEN** Returns validation result with errors for invalid structure
